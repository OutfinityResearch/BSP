Wiki & Glossary

Definitions of technical terms and concepts used within the BSP project.

Data Structures

Bitset

Core

A data structure that compactly stores a set of integers. In BSP, bitsets represent the presence of features (identities). Operations like intersection (AND) and union (OR) are extremely fast.

Implementation: This repository uses an in-repo implementation (SimpleBitset) described in the Data Structures spec (DS-002).

Identity

Concept

An atomic feature in the system, represented by a unique integer ID (0 to N-1). Identities can map to tokens, n-grams, or other input features.

Group

Core

Also referred to as a "Concept". A Group is a learned collection of Identities that frequently appear together. It is the fundamental unit of meaning in BSP, analogous to a stable embedding vector in neural networks.

Learning & Metrics

Surprise

Metric

The difference between the actual input and what the system's active Groups can explain. Formally: Input \ Reconstruction. High surprise drives learning (creating new groups or updating existing ones).

Hallucination

Metric

Bits that the active Groups predict (reconstruct) but are NOT present in the actual input. Formally: Reconstruction \ Input. This represents "noise" or over-generalization.

Salience

RL

A property of a Group (0.0 to 1.0) indicating its long-term value or utility. Groups that contribute to positive rewards accumulate Salience, making them "stickier" and easier to activate.

Importance

RL

A transient property of an event/interaction. Calculated based on Novelty (Surprise), Utility (Reward), and Stability. High importance events trigger stronger learning updates.

Architecture

Component

GroupStore

Component

The database of all learned Groups. It supports operations like creating, updating, merging, and pruning groups.

SimpleBitset

Implementation

A minimal bitset implementation backed by Uint32Array. It supports fast membership checks and bitwise operations without external dependencies and is used by the current Node.js implementation of BSP.

Theory

Predictive Coding

Theory

A theory of cognition where the brain is constantly generating predictions about sensory input. BSP implements this by using previous context to predict the next state and learning only from the error (surprise).

MDL (Minimum Description Length)

Math

A principle stating that the best model is the one that compresses the data the most. BSP uses an MDL-inspired objective function: minimizing the sum of Unexplained Bits (Surprise) + Model Cost (Hallucination + Number of Groups).
