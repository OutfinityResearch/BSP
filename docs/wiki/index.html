Wiki & Glossary

Definitions of core concepts used in the BPCM project.

Bitset / Bitmap

A data structure that represents a set of integers as a sequence of bits. If the 5th bit is 1, it means the integer 5 is in the set.

In BPCM: Used to represent inputs and concepts efficiently. Instead of storing an array `[1, 5, 10]`, we store a bitset where indices 1, 5, and 10 are set to 1.

SDR (Sparse Distributed Representation)

A way of representing information where only a small percentage of bits are active (1) at any time. This mimics how neurons work in the brain.

Benefit: Robust to noise. Even if you flip a few bits, the overall meaning (the set overlap) remains largely the same.

Jaccard Index

A statistic used for comparing the similarity and diversity of sample sets.

J(A, B) = |A ∩ B| / |A ∪ B|

In BPCM: We use this to decide if two Groups are similar enough to be merged. If two concepts share 90% of their bits, they are likely the same concept.

Popcount (Population Count)

The number of set bits (1s) in a bitset. Also known as Hamming weight.

Example: Popcount of 10110 is 3.

Surprise

The information in the input that the model failed to predict.

Surprise = Input AND (NOT Prediction)

If Surprise is 0, the model perfectly understood and predicted the input.

Hallucination

Information that the model predicted, but was not present in the actual input.

Hallucination = Prediction AND (NOT Input)

While often bad, small hallucinations in BPCM can represent "imagination" or filling in missing details.
