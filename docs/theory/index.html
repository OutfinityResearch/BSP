Theory of Operation

BSP is grounded in the Predictive Coding theory of cognition, which postulates that the brain is constantly generating predictions about sensory input and only processing (learning from) the "surprise" or prediction error.

The Learning Loop

Unlike backpropagation which minimizes a global loss function across a fixed network, BSP performs local, incremental updates based on how well existing "Groups" (concepts) can explain the current input.

Minimum Description Length (MDL)

The core objective function is inspired by MDL. The goal is to transmit the input x using a set of active groups A with minimum cost.

The Objective Function

L(x, A) = |Surprise| + β * |Hallucination| + γ * |A|
  • Surprise: Bits in input NOT covered by groups (x \ reconstruct(A)). Must be transmitted raw.
  • Hallucination: Bits in groups NOT in input (reconstruct(A) \ x). Represents error/noise.
  • |A|: Model cost. Using fewer groups is better (Occam's razor).

Reinforcement Learning (RL)

BSP integrates RL not as a separate fine-tuning step, but as a continuous modulation of the learning process. The "Importance" of a signal determines how aggressively the system updates its internal state.

Importance & Salience

Two key metrics drive the "value" system of BSP:

  • Importance (Episodic): Calculated per interaction. Combines Novelty (how surprising), Utility (reward), and Stability. High importance events trigger stronger learning updates and are prioritized in the Replay Buffer.
  • Salience (Structural): A property of a Group. Represents the long-term utility of a concept. Groups that frequently lead to high rewards accumulate Salience, making them more likely to be activated in ambiguous situations.

For detailed algorithms, see DS-003: Learning Algorithms and DS-005: RL & Importance.
