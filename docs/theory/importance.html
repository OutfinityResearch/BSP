Importance & Salience

BSP differentiates between the value of an Event and the value of a Concept.

Importance (Episodic Value)

Importance is a property of a specific moment in time (an interaction). It determines "how much we should learn from this".

Importance = f(Novelty, Utility, Stability)

  • Novelty: How surprised was the system? (High Surprise = High Learning Opportunity)
  • Utility: Did this lead to a reward?
  • Stability: Is this pattern consistent over time?

Salience (Structural Value)

Salience is a property of a Group (Concept). It represents the long-term usefulness of that concept.

Groups with high Salience:

  • Are activated more easily (lower threshold).
  • Are less likely to be pruned/forgotten.
  • Are prioritized during prediction.

Dynamic Memory Management

Because BSP runs on constrained memory (unlike static LLM weights), it must constantly decide what to keep and what to forget. Salience provides the metric for this "Garbage Collection". Low-salience, rarely used groups are pruned to make space for new learning.
