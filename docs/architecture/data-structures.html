Data Structures

BSP is optimized for CPU performance using specialized data structures. The primary challenge in sparse systems is memory efficiency and intersection speed.

Bitsets & Roaring Bitmaps

Instead of dense float vectors, BSP uses Roaring Bitmaps. A bitset represents a set of integers (Identity IDs).

Why Bitsets?

Classic ML uses embeddings (e.g., [0.1, -0.5, 0.9, ...]). Operations are O(N) multiplications.

BSP uses sets (e.g., {10, 55, 1009}). Intersection is very fast. Roaring Bitmaps compress these sets, often requiring only a few bytes for sparse data.

Identity Universe

Every unique feature (token, n-gram, pixel hash) is mapped to an integer ID from 0 to N (e.g., 1,000,000). This is the "Identity Universe".

The Group Object

A "Group" is the fundamental unit of memory. It is a reified concept.

interface Group {
  id: number;                          // Unique identifier
  
  // Membership
  members: RoaringBitmap;              // The "essence" of the group
  memberCounts: Map;   // How often each feature appeared
  
  // Metadata
  salience: number;                    // Importance (0-1)
  age: number;                         // Epochs since creation
  lastUsed: number;                    // Timestamp last activation
}

Inverted Index (BitmapIndex)

To find groups quickly, we use an inverted index. For every Identity ID, we know which Groups contain it.

Example: if token "Cat" is ID 55, Index[55] = {GroupA, GroupB}. When "Cat" appears in input, we immediately know GroupA and GroupB are candidates.
